## Goal
Make every sub-agent role (`/plan /verify /triage /monitor /report /research`) usable three ways, and fix a model-spec bug:
1. **Tool-mode (the reported bug):** the *running model* can invoke any role as an isolated sub-agent **mid-turn**, seeing its summary in-context.
2. **Command-mode (unchanged):** the user keeps typing `/<role> …`.
3. **`/<role>-main`:** the user chats with the **main** session running **under that role's methodology** (the four roles whose tools live in the main session; monitor/research route to the isolated sub-agent — see 4b).
4. **Model-spec fix:** all models resolve through the **GitHub Copilot** login, never direct Anthropic/OpenAI.

## Root-cause diagnosis (confirmed by review, evidence-backed)
- **Roles are model-unreachable.** `subagents/index.ts` has six `registerCommand` (`:556,665,783,884,993,1123`) and **zero** `registerTool`; commands are user-input-only (`types.d.ts:793-799`; `_tryExecuteExtensionCommand` reached only from the `prompt()` text path, `agent-session.js:708-715`). The model emits only tool calls.
- **`nextTurn` results never reach the live turn.** Handlers post via `pi.sendMessage(...,{deliverAs:"nextTurn"})`; queued to `_pendingNextTurnMessages`, injected only at the next prompt (`agent-session.js:790-794,985-986`); "does not interrupt or trigger anything" (`docs/extensions.md:1334`).

## Approach (reuse ladder — thin wrappers over the unchanged engine)
Reused unchanged: `runSubagent`, `runner.ts` allowlists, `runFixedTee` redaction, `--no-extensions` isolation, `fileSig` fallback, `extractSummary`, `handoff*`, `computeDiff/computeGitLog`, `findRepoRoot`. No runner changes (4b routes to `runSubagent`, so the #10 `registerRunExperiment` refactor is avoided — see 4b).

### A) Tool-mode (model invokes a role mid-turn)
- Refactor each handler body into a shared `runXRole(input, ctx)` (mechanical; no behavior change). Template: `examples/extensions/subagent/index.ts`. **`runXRole` is parameterized per role, not uniform — monitor/report carry extra orchestration (see F5).**
- **Tool names — namespaced, mandatory (R4-1):** register the six wrappers as `subagent_plan`, `subagent_verify`, `subagent_triage`, `subagent_monitor`, `subagent_report`, `subagent_research`. The `subagent_` prefix avoids collision with built-ins (`read/write/edit/bash/grep/find/ls`) and extension tools (`run_check/run_experiment`): pi's tool registry is **last-write-wins by name with NO collision error** (`agent-session.js:1873-1876,1846-1851`), so a wrapper reusing a built-in name would **silently shadow it session-wide**. Parameter schemas (mirror the command arg parsing): `subagent_plan {feature, task}`; `subagent_verify {feature?, note?}`; `subagent_triage {logPath?, note}`; `subagent_monitor {experiment, note?}`; `subagent_report {subject, audience?, sources?}`; `subagent_research {topic, question}`.
- Six `pi.registerTool` wrappers; each `execute(id, params, signal, onUpdate, ctx)`: `findRepoRoot(ctx.cwd)` → build the same `userTurn` → `runSubagent({…per-role tools/model/runnerPath…})` → same `fileSig`/`wroteItself` fallback → **return `extractSummary(res.finalText,10)` as content**.
- Hard requirements (confirmed #3): `label` required (`types.d.ts:339`); `executionMode:'sequential'` (`:359`) — serializes a *batch* but does **not** cap *count* (`agent-loop.js:262-281`, N3); add a **wall-clock timeout + `SIGTERM→SIGKILL`** and pass `signal` (runSubagent has none today, `:375-427`); forward `onProgress→onUpdate`; return **only** the ≤10-line summary (artifact stays on disk — protects main context).
- **Error surfacing — THROW, settled (N1):** a *returned* `isError:true` is **inert** — `agent-loop.js:433` hardcodes `isError:false` on the no-throw path; only the `catch` sets it (docs: "Returning a value never sets the error flag… throw an error from execute", `extensions.md:~1809`). So `execute()` must **throw a controlled, redaction-safe `Error(<short reason>)`** on failure — the thrown message becomes the model-visible content. Aside: `runner.ts`'s `isError:true` returns for `run_check`/`run_experiment` refusals (`:87,116,202`) are therefore **cosmetic** — pre-existing, out of scope.
- **Prompt footprint vs discoverability (N6 + gotcha — refined):** `setActiveToolsByName` rebuilds the prompt from active tool *names* only (`agent-session.js:543-557,647`), so omitting prose adds zero tokens — BUT a tool needs a **`promptSnippet`** to appear in the human-readable "Available tools" list, and discoverability is the whole point of tool-mode. Decision: give each role tool a **one-line `promptSnippet`** (tiny, deliberate cost so the model reliably reaches for it) and **skip the bulkier `promptGuidelines`**. (This walks back N6's "omit `promptSnippet`": zero-bloat isn't worth an undiscoverable tool.)
- **No orphaned subprocesses on `/reload`/quit (gotcha):** `ctx.signal` covers turn *abort* (Esc) only — NOT shutdown — and the harness has **no `session_shutdown` handler** today. A role tool's `execute` spawns a child *during streaming*, so track live children in a module-level `Set` and add a `session_shutdown` handler that `SIGTERM→SIGKILL`s them; otherwise `/reload`/quit orphans the `pi` subprocess. (Reuse the same kill path as the wall-clock timeout.)
- **`subagent_monitor` is NOT a thin pass-through (F5):** like the command handler (`index.ts:884-991`), its `execute()` must generate a slug-valid `runId` (`^[A-Za-z0-9._-]{1,80}$`), compute `reportPath`/`logRel`, build the `handoffMonitor(...)` user turn, validate the free-text `experiment` in-handler against `listExperiments` (N7; throw on unknown), then call `runSubagent` — else the sub-agent's `run_experiment({runId})` is refused (`runner.ts:158`). (`mkdir memory/runs/` is optional — `run_experiment` creates the log dir at `runner.ts:174`.)
- **Placement + result-redaction notes (gotchas):** keep the tool-mode + `-main` code in the **existing `subagents/` extension dir** (no new dir → no `install.sh` change; `/reload` can't load a never-installed dir). The role tool's result is scrubbed by `secret-redaction` only in its **`content`** (the summary), not `details` (which crosses back raw) — so keep `details` free of anything sensitive.
- Safety preserved byte-for-byte (confirmed #4): child keeps `--no-extensions/-nc/--no-skills/--no-prompt-templates/--no-themes/--no-session` + per-role `--tools` allowlist; execution funnels through `run_check`/`run_experiment`; redaction at `runFixedTee`; role tool not re-loaded in child (no recursion).

### B) Gating — **auto-judge only** (owner decision)
- Add the six exact tool names (`subagent_plan`, `subagent_verify`, `subagent_triage`, `subagent_monitor`, `subagent_report`, `subagent_research`) to `autoJudge.guardedTools` in `harness/checks.json`. That gate is **exact-match** (`Array.includes`, `auto-judge/index.ts:300`) — a typo or namespace drift silently un-gates the tool, so the strings must match the registered names verbatim. When `/autojudge on`, each model-driven spawn is judged ALLOW/DENY (`:299-300`). Right policy gate — `command-guard` matches only `bash/shell/write/edit` (`command-guard/index.ts:82,93`) and won't gate a custom tool name. Gate composition is safe (first-block-wins, `agent-loop.js:386-391`, N5).
- **Keep `autoJudge.contextDiff:false` for the role tools (N8):** arming auto-judge already ≈ doubles spawns (a judge subprocess per call); adding a working-tree git diff per judgement compounds cost for no safety gain here.
- Honest note (accepted): auto-judge is opt-in / default-OFF (`:269`), and there is **no hard count/concurrency cap** — so when not armed, model-driven role spawns are bounded only by `executionMode:'sequential'` (one at a time) and the model's own judgment. Accepted tradeoff (children are shell-free, edit-free, `--no-extensions`).

### C) `/<role>-main` (main session under the role)
- Single-slot `activeRole: RoleName | null` (replace, never stack). Commands `/<role>-main on|off`. Handler sets/clears `activeRole`, snapshots `pi.getActiveTools()` on entry, `pi.setActiveTools(ROLE_MAIN_TOOLS[role])`, restores on `off`. Template: `preset.ts` + `plan-mode/index.ts`.
- **`before_agent_start` appends `roleBody(activeRole)` = ONLY `harness/prompts/<role>.md`** (the per-role methodology body) to `e.systemPrompt`; auto-reverts to base when null (`agent-session.js:810-817`). **Do NOT re-append the `AGENTS.md` brief here (F1):** the main session runs without `-nc`, so `AGENTS.md` is already auto-loaded into the base prompt (`usage.md:96-102`; visible as `systemPromptOptions.contextFiles`, `extensions.md:503`) — re-injecting duplicates it. (Differs from `runSubagent`, which uses `-nc` and must inject brief+body itself.) Confirmed #5.
- **Strip the isolated-sub-agent terminal contract from injected bodies (F2, scoped):** `triage.md` (`:31,34-36`) and `report.md` (`:35-38,43-44`) embed "write exactly ONE file" + "final message must be `## SUMMARY` … Nothing else after it" + "the harness surfaces only the SUMMARY" **in the body** — injected into an interactive chat that would truncate `triage-main`/`report-main` replies to a SUMMARY stub and nudge a stray `memory/*.md` write. `plan.md` and `verify-change.md` are **already clean** (their contract lives only in the `handoff*` builders, which `-main` doesn't inject — plan-main/verify-main need no change). Fix: move that contract out of `triage.md`/`report.md` bodies into their `handoff*` builders (mirroring plan/verify), OR have `roleBody()` filter it. `monitor.md`/`research.md` keep their in-body contract — they're 4b (isolated sub-agent), where it's correct.
- **Tool clamp is REQUIRED** + a `tool_call` block-gate while `activeRole` set (`plan-mode/index.ts:82-89,122-132`):
  - `plan-main`/`report-main` → read,grep,find,ls,write · `verify-main`/`triage-main` → read,grep,find,ls,run_check (NO edit/bash).
  - *(F3, accepted by-design):* `write` for plan/report-main is intentional — their artifacts live under `memory/`, which AGENTS.md treats as writable (not a Boundary); the clamp only *narrows* the default `read,bash,edit,write` session. Any new protected path belongs in `command-guard` boundaries, not a `-main`-specific clamp.
- `ROLE_MAIN_TOOLS` deliberately **excludes** the slice-1 role tools (the model shouldn't spawn sub-agents *while it is* the role). The `getActiveTools()` snapshot now contains those six, so **restore-on-off must preserve them**.
- **Reload/resume must not strand the user in a silent half-state (N2, MAJOR):** on `/reload` the runtime is rebuilt with `activeToolNames: getActiveToolNames()` (`agent-session.js:1943`) so the **clamp survives**, but the extension re-instantiates → in-memory `activeRole` resets to `null` → `before_agent_start` reverts the prompt to base (`:816`): the user is left **clamped read-only with the role + status gone**. On a fresh build/resume the default set is the hardcoded `["read","bash","edit","write"]` (`:1926-1929`) → the **opposite** desync. FIX: **persist `activeRole` via `pi.appendEntry` and restore it on `session_start`** (re-apply clamp + re-arm `before_agent_start` + status), per `plan-mode/index.ts:91-97,291-339`; PLUS a safety net — whenever `activeRole` resolves to `null` on `session_start`, explicitly `setActiveTools(<normal full set>)` so a clamp can never outlive its role.

#### Slice 4 split (review #9/#10 resolved by owner choosing (ii))
- **4a (in-session mode): plan / verify / triage / report-main.** Tools already in the main session — the `before_agent_start` (body-only) + tool-clamp path above. triage/report bodies need the F2 strip.
- **4b: monitor-main / research-main → spawn the isolated sub-agent (option ii).** Their real tools (`run_experiment`, web search) are subprocess-only; rather than register privileged tools into the full-tools main session, `/monitor-main` and `/research-main` invoke the **isolated sub-agent** (reuse `runXRole`/`runSubagent`) and return the summary. Routing through the **subprocess moots #9** (no `run_experiment` in the main session to gate) and **#10** (no `registerRunExperiment` export / `ctx.cwd` refactor) and keeps full isolation + `runFixedTee` redaction. Net: real in-session `-main` is the four clean roles; monitor/research-`main` are isolated sub-agents.

## Risks
- **Reload/resume `-main` desync (N2)** is the highest-risk lifecycle bug — persist+restore + null-safety-net is mandatory.
- **Tool-name shadowing (R4-1)** is silent (last-write-wins, no error) — the `subagent_` prefix is mandatory, not cosmetic; `guardedTools` must match exactly.
- Model-driven fan-out has no hard count cap (owner-accepted) — bounded only by `sequential` + an armed auto-judge.
- Tool result re-enters the same turn → summary-only is load-bearing for context budget.
- `-main` clamp + `tool_call` gate are load-bearing (role restrictions are otherwise advisory).
